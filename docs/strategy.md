# Awtad Energy Website Strategy

## Project Summary
Awtad Energy is an original Oman-based renewable infrastructure brand concept. The website is intentionally positioned away from a generic “green company” template and toward a more premium Gulf-market language built around infrastructure credibility, regional relevance and long-horizon performance.

The structure starts from operating reality, then moves into platforms, delivery, project models and editorial depth. That sequence makes the site feel more commercially serious and more useful for utilities, industrial buyers, partners and institutional stakeholders.

## 1. Information Architecture

### Primary Navigation
| Page | Purpose | Primary Audience | Primary CTA |
| --- | --- | --- | --- |
| Home | Establish brand thesis, credibility and platform range | First-time visitors, partners, investors | Start a Brief |
| Platforms | Explain solar, wind, hybrid, battery and modular systems as deployable platforms | Technical buyers, project sponsors | Brief a Project |
| Delivery | Show advisory, engineering, deployment and O&M as one execution chain | Buyers comparing delivery maturity | Request a Discussion |
| Projects | Present representative application typologies without fake proof | Prospects evaluating fit | Compare Your Brief |
| Journal | Add editorial authority and long-term SEO value | Investors, partners, technical readers | Discuss a Brief |
| Company | Define positioning, standards and operating principles | Partners, institutional stakeholders, recruits | Start a Brief |
| Contact | Convert interest into a structured project enquiry | High-intent leads | Send Project Brief |

### Secondary Information Architecture
| Path | Role |
| --- | --- |
| Platform subsections | Solar Fields, Wind Corridors, Hybrid Microgrids, Battery Reserve, Modular Power Blocks |
| Sector lenses | Utilities and IPPs, Ports and Logistics, Industrial Sites, Remote Assets, Public Infrastructure |
| Editorial lanes | Infrastructure Strategy, Field Engineering, Asset Performance |
| Conversion paths | Start a Brief, Compare Your Brief, Discuss a Similar Program |
| Future expansion | Arabic version, Careers, ESG/Standards page, downloadable briefs, case-study CMS |

### User Journey Logic
1. First-time visitor lands on `Home` and understands the brand thesis in under 20 seconds.
2. Interested buyer moves to `Platforms` or `Delivery` depending on whether they are comparing technologies or execution models.
3. Prospect validates relevance on `Projects`.
4. Higher-intent reader uses `Journal` or `Company` to test maturity and seriousness.
5. Serious lead reaches `Contact` via “Start a Brief” rather than a passive “contact us” prompt.

### Why This IA Is Original
- It starts with operating conditions and platform logic instead of a generic services-first checklist.
- It treats consultancy and O&M as part of one delivery architecture rather than isolated menu items.
- It uses representative project typologies instead of fabricated numeric case studies.
- It gives editorial space real strategic value rather than burying insight content in a blog afterthought.

## 2. Brand Direction

### Name
**Awtad Energy**

“Awtad” references stability and anchoring. That suits a renewable infrastructure business positioned around dependable long-term performance.

### Positioning Statement
Awtad Energy is an Oman-rooted renewable infrastructure partner delivering solar, wind, hybrid, storage and modular energy systems for complex operating environments across the Gulf.

### Brand Promise
**Clean power that behaves like infrastructure.**

This line avoids eco-cliche language and immediately tells the market that the company is focused on disciplined execution, reliability and asset seriousness.

### Audience Priorities
| Audience | What they need from the site |
| --- | --- |
| Utilities and IPPs | Seriousness, platform range, execution credibility |
| Industrial and logistics operators | Uptime logic, hybrid/storage clarity, retrofit practicality |
| Government and public infrastructure stakeholders | Stability, standards, regional fluency |
| Investors and partners | Clear positioning, premium brand cues, long-horizon thinking |
| Technical consultants and OEM partners | Integration discipline and delivery maturity |

### Voice and Messaging Rules
| Attribute | Direction |
| --- | --- |
| Tone | Calm, exacting, commercially literate |
| Sentence style | Short to medium, decisive, low-hype |
| Vocabulary | Infrastructure, uptime, dispatch, resilience, delivery, lifecycle, service response |
| Avoid | Startup slogans, activist language, overclaiming, vague “green future” copy |
| Emotional register | Quiet confidence rather than loud innovation theatre |

### Visual System
| Element | Direction |
| --- | --- |
| Core palette | Sea teal, deep slate, sun metal, copper heat, limestone neutrals |
| Brand metaphor | Coastline to grid edge |
| Layout feel | Editorial plus industrial, not generic corporate grid |
| Shapes | Rounded infrastructure cards, contour fields, layered horizons, modular blocks |
| Motion | Reveal-on-scroll, panel lift, restrained hover states |
| Typography | `League Spartan` for headings, `Manrope` for body, `IBM Plex Sans Arabic` for regional accents |
| Surface logic | Soft warm backgrounds with darker infrastructure bands for high-contrast sections |

### Design Principles
1. Start with conditions, not technology.
2. Make visuals feel commissioned, even when abstract.
3. Use architecture and sequencing to imply competence.
4. Keep CTAs high-intent and specific.
5. Let editorial depth reinforce premium positioning.

## 3. Homepage Wireframe

### Homepage Flow
1. Sticky header with compact navigation and persistent `Start a Brief` CTA.
2. Hero with brand thesis on the left, three tall image panels on the right and a proof panel underneath.
3. Operating Reality section with four climate/terrain cards and one large terrain canvas.
4. Platform Mix section with five platform cards.
5. Delivery Architecture section in a dark band with four execution steps.
6. Priority Sectors section with five sector cards.
7. Program Models section with three representative project stories.
8. Field Notes section with three editorial teasers.
9. Final CTA banner leading to the project-brief page.
10. Footer with core links, focus areas and contact route.

## 5. Image Direction For Every Section

### General Art Direction
- Use large, art-directed imagery rather than small decorative thumbnails.
- Avoid generic stock with smiling workers, handshake scenes or pure panel closeups.
- Favor wide cinematic compositions, industrial landscapes, modular details and environmental context.
- Use atmospheric light: dawn, late afternoon, heat haze, marine light, low-angle sun.
- Keep people secondary unless scale or inspection activity is needed.

### Home
| Section | Image Direction |
| --- | --- |
| Hero | Triptych of solar plane, wind field and battery architecture. Wide, cinematic, high negative space, strong sky and terrain contrast. |
| Operating Reality | One large aerial-inspired terrain composition showing coast, interior and industrial geometry; four supporting abstract climate cards. |
| Platform Mix | Distinct visual language per platform: panel geometry, wind silhouette, dispatch grid, battery cell rhythm, modular container stacks. |

### Production Photography and Image Prompt Notes
1. Commission a custom image set if possible. Prioritize Oman-specific landscapes, ports, industrial edges and renewable infrastructure in climate-credible conditions.
2. Shoot wide horizontal masters first, then derive vertical crops for homepage triptychs and card layouts.
3. Capture modular equipment with clear scale cues.
4. Maintain a consistent grade: warm highlights, deep teals, restrained saturation, no neon greens.
5. Build an image prompt library for solar, wind, storage, modular deployment, field engineering and coastal industrial contexts.

## 6. CTA Strategy

### CTA Hierarchy
| Level | CTA | Purpose |
| --- | --- | --- |
| Primary | Start a Brief | Main sitewide conversion action |
| Secondary | Explore Platforms / View Delivery Model / See Project Models | Keep users moving through evaluation |
| Page-specific | Compare Your Brief / Discuss a Similar Program / Request a Discussion | Contextual next-step CTAs tied to page intent |
| Tertiary | Read Field Notes | Lower-friction nurturing path |

### CTA Principles
1. Use intent-rich verbs: start, compare, discuss, brief.
2. Avoid “Learn More” as a core CTA.
3. Keep the primary CTA visible in the sticky header.
4. End every major page with a conversion banner.
5. Match CTA wording to the page mindset:
   - Platforms = “Brief a Project”
   - Delivery = “Request a Discussion”
   - Projects = “Compare Your Brief”
   - Journal = “Discuss a Brief”
   - Contact = “Send Project Brief”

### Lead Quality Strategy
- The site is optimized for better-fit B2B leads rather than maximum low-intent submissions.
- Contact language assumes the user has a real site, load profile or project question.
- Form fields collect stage and need, which improves downstream qualification.

## 7. Full Front-End Implementation Plan

### Current Prototype Build
| Area | Implemented |
| --- | --- |
| Page set | `index.html`, `platforms.html`, `delivery.html`, `projects.html`, `journal.html`, `about.html`, `contact.html` |
| Styling | Shared design system in `assets/styles.css` |
| Interactions | Navigation toggle, reveal animations, active nav state, contact-form prototype state in `assets/app.js` |
| Documentation | Strategy and IA plan in `docs/strategy.md` |

### Recommended Production Path
1. **Framework decision**
   - Keep static HTML/CSS/JS if the site will remain low-volume and brochure-like.
   - Move to Astro or Next.js if multi-author CMS workflows, localization or dynamic case studies are needed.
2. **Content model**
   - Collections: pages, platforms, project typologies, articles, CTA blocks, settings.
   - Reusable fields: eyebrow, headline, intro, image asset, CTA label, CTA link, tags.
3. **Design system**
   - Preserve the current token set for color, type, spacing and radius.
   - Convert repeating patterns into components: header, hero, visual card, CTA banner, footer, form.
4. **Image pipeline**
   - Replace abstract concept visuals with commissioned photography or generated art-directed image assets.
   - Create desktop, tablet and mobile crops for every hero and high-impact card.
   - Use next-gen formats and lazy loading.
5. **Localization**
   - Plan for bilingual English/Arabic support.
   - Mirror layout logic where needed for RTL.
   - Keep Arabic font pairing ready at component level.
6. **Lead capture**
   - Connect the current form to HubSpot, Salesforce or a secure email endpoint.
   - Add success states, spam control and event tracking.
7. **Accessibility**
   - Maintain semantic headings and keyboard navigation.
   - Test contrast and focus states after real imagery is introduced.
   - Add alt text and descriptive labels for production images.
8. **Performance**
   - Compress hero media.
   - Avoid loading all images above the fold.
   - Keep web fonts subsetted and preconnected.
9. **Analytics**
   - Track header CTA clicks, page-end CTA clicks, form starts, form submits and article-to-contact journeys.
10. **QA**
    - Test responsive layouts from 320px upward.
    - Validate navigation on touch devices.
    - Run Lighthouse and accessibility checks after image replacement.

| Phase 2 | Real image production, CMS integration, final copy editing, analytics and CRM wiring |
